ScholarDesk — Product Specification
=====================================

Overview
--------
A single-file academic todo/task manager targeting students. Built as a
ScholarDesk-style dashboard with rich task organization (due dates, priorities,
subjects/categories, subtasks), a fixed sidebar, stat cards, quick-add bar,
and a bottom performance widget. Persists data via localStorage.


Users
-----
Academic users managing coursework, assignments, and personal tasks from a
single browser tab.


Design System
-------------
Follows the "Digital Scholar" design system:
- Fonts: Manrope (headlines), Inter (body) — loaded from Google Fonts
- Primary color: #004ac6 (academic blue)
- No 1px dividers — hierarchy via tonal surface shifts (background → surface-low
  → surface-lowest → surface-high)
- Buttons: linear gradient (#004ac6 → #2563eb) at 135°, full pill roundedness
- Inputs: surface-low background, ghost blue focus ring at 20% opacity
- Sidebar: glassmorphism (rgba 80% + backdrop-filter blur)
- Shadows only on modal overlays


Features
--------

1. Task Management
   - Add tasks via modal or quick-add bar (press Enter)
   - Edit and delete tasks
   - Mark tasks complete/incomplete via checkbox
   - Clear all completed tasks

2. Due Dates
   - Optional due date per task (date picker)
   - Visual indicators on card:
       - Overdue   — red chip
       - Due today — amber chip
       - Upcoming  — grey chip
   - Sortable by due date

3. Priority Levels
   - Three levels: High / Medium / Low
   - Material-style container badges (error-container / secondary-container / surface-variant)
   - Filterable and sortable by priority

4. Subjects (Categories)
   - Each task belongs to one subject (e.g., Mathematics, Physics, Personal)
   - User can create, rename, and delete subjects
   - Sidebar lists subjects with colored dot indicators
   - Subject shown as a colored chip on each task card

5. Subtasks
   - Each task can have multiple subtasks
   - Subtasks are individually checkable
   - Progress shown as "X/Y" count + mini progress bar on the parent card
   - Progress bar turns green when all subtasks are complete
   - Parent task auto-completes when all subtasks are done (with confirmation)

6. Dashboard Stat Cards
   Four cards at the top of the main area:

   Card         | Content
   -------------|------------------------------------------
   Total Active | Count of all incomplete tasks
   Due Today    | Count of incomplete tasks due today
   Overdue      | Count of incomplete overdue tasks (red bg)
   Completed    | Count of tasks completed this week

7. Bottom Widgets
   - Weekly Performance: gradient card showing completed-this-week count
   - Scholar Tip: static motivational card with hashtag tags

---------

  +------------------+------------------------------------------+
  |  Sidebar (fixed) |  Main Content                            |
  |                  |                                          |
  |  ScholarDesk     |  Today's Focus          [search] [🔔][🕐]|

  |  MAIN NAVIGATION |  [Total Active][Due Today][Overdue][Done]|
  |  - Dashboard     |                                          |

  |                  |                                          |
  |  SUBJECTS        |  Urgent & Active           N items       |
  |  • Mathematics   |  +------------------------------------+  |

  |  + New Subject   |  |      Subject • Date • Subtasks X/Y|  |

Each task card shows:
  Left 4px border (color = status) · checkbox · title · status badge ·
  subject chip · due date chip · priority badge · subtask progress ·
  edit / expand / delete buttons

Clicking expand (▼) reveals subtask list and add-subtask form.

----------------

Status    | Left Border | Badge
----------|-------------|--------

Today     | Amber       | TODAY (amber)
Upcoming  | Blue        | UPCOMING (blue)

Data Model
----------

{
  "categories": [
    { "id": 1713000000000, "name": "Mathematics", "color": "#004ac6" }
  ],
  "todos": [
    {
      "id": 1713000000001,
      "text": "Submit Quantum Mechanics Thesis Draft",
      "done": false,
      "priority": "high",
      "dueDate": "2026-04-01",
      "categoryId": 1713000000000,
      "subtasks": [
        { "id": 1713000000002, "text": "Review references", "done": true },
        { "id": 1713000000003, "text": "Write conclusion", "done": false }
      ],
      "createdAt": "2026-03-29",
      "completedAt": null
    }
  ]
}

Stored under a single "todos-v2" key in localStorage.


Sort & Filter
-------------

Control             | Options
--------------------|------------------------------------------
Filter by status    | All / Active / Done (cycled via pill click)
Filter by priority  | All / High / Medium / Low (cycled)
Filter by due       | All / Today / Overdue / This Week (cycled)
Filter by subject   | All / [each subject] (dropdown)
Sort by             | Date Added / Due Date / Priority (dropdown)

Quick-add bar also accepts Enter key to add a task with no metadata.


Tech Stack
----------
- Single HTML file (HTML + CSS + vanilla JS, ~900 lines)
- Google Fonts: Manrope + Inter
- No build tools, no dependencies
- localStorage for persistence
- Source: GitHub (asafzus/todo-list-app)
- Deployed: Netlify (auto-deploy on push to master)


Out of Scope
------------
- User accounts / login
- Backend or database
- Collaboration / sharing
- Mobile app
- LLM integration
